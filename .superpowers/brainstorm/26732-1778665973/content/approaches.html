三种改造方向

选一种你倾向的方向，我会基于它设计具体方案。

A

原地打补丁

在现有两个 tab 内各加若干图表：JBStart 加 yield% 计算和 slot 趋势，yield monitor 加 DUT 分布和时间趋势。不动框架结构。

API目录 | yield monitor | JB START | 表浏览
现有筛选区（不变）
现有图表（保留）
✚ yield% 条形图
✚ DUT 触发分布
✚ 时间趋势

优点

  • 改动最小，风险低

缺点

  • 3个查询按钮的问题没解决
  • 页面越来越长，难找信息
B

新增专项分析 tab（推荐）

保留现有两个 tab 不动，新增两个专项 tab：根因排查 和 批次汇总，每个 tab 一键查询、图表从全局到细节分层展示。

API目录 | yield monitor | JB START | 根因排查 ✚ | 批次汇总 ✚ | 表浏览
根因排查 tab
① Lot+Device 筛选
② 各 Slot 不良 die 条形图
③ Bin 分布 + yield% 数字
④ ProbeCard 对比
⑤ DUT 触发热度
批次汇总 tab
① Device + 时间范围
② 各 Lot yield% 排名
③ 趋势折线（按天）
④ 探针卡良率排名
⑤ 触发类型分布

优点

  • 不破坏现有功能
  • 用途明确，一键查询
  • DUT/probecard 成为核心维度

缺点

  • tab 数量增加到6个
C

重建两个主 tab

把 yield monitor 和 JB START 两个 tab 推倒重来，单一「查询」按钮，结果分层呈现：汇总指标卡 → 主图 → 下钻图表，probe card 和 DUT 成为一级分析维度。

  • 指标卡一眼看到关键数字

缺点

  • 旧功能要移植确保不丢
