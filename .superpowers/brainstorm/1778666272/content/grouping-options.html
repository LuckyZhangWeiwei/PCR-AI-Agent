分组 & 嵌套分组 — 三种方案

现有 v3 aggregate API 支持任意多维 groupBy，问题是：如何在 UI 上呈现"组内再分组"？

A

多维平铺表（最简单）

用户勾选多个维度，一次请求返回多列扁平结果。
示例：groupBy = lot + cardId + bin → 每行同时含 LOT、探针卡、BIN 三列

优点

  • 实现最简单，一次 API 请求
  • 支持无限维度组合
  • 可导出 CSV

缺点

  • 行数多时看不清层级关系
  • 同一 LOT 会重复出现多行
B

两级下钻（推荐）

上方展示第一级分组结果；点击某行 → 弹出/展开第二级细分。
两次独立 API 请求，第二次自动带入点击行的筛选值。

优点

  • 层级清晰，符合调查习惯
  • 第二级维度可按需切换（cardId / slot / bin）
  • 复用现有 v3 aggregate API，不需改后端

缺点

  • 两次请求（不过是并行或按需触发）
  • 只支持两层（满足大多数场景）
C

折叠树表（最强大）

一次请求拉取多维度聚合数据，前端自动按选择的"主维度"分组折叠； 点击展开可见子行。支持任意层级。

优点

  • 层级最直观，一屏看全所有 Lot+Card+Bin
  • 点击折叠/展开，不需要跳页
  • 支持 3 层甚至更多

缺点

  • 前端实现最复杂（树表组件）
  • 数据量大时行数可能很多
