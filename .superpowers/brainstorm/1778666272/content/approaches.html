三种改造方向

选一种你倾向的方向，我会基于它设计具体方案。

A

原地打补丁

在现有两个 tab 内各加若干图表：JBStart 加 yield% 计算和 slot 趋势，yield monitor 加 DUT 分布和时间趋势。不动框架结构。

API目录 | yield monitor | JB START | 表浏览
现有筛选区（不变）
现有图表（保留）
✚ yield% 条形图
✚ DUT 触发分布
✚ 时间趋势

优点

  • 改动最小，风险低

缺点

  • 3个查询按钮的问题没解决
  • 页面越来越长，难找信息
B

重建两个主 tab（推荐）

把 yield monitor 和 JB START 两个 tab 推倒重来。单一「查询」按钮，结果分三层：汇总指标卡 → 主分析图 → 明细下钻，ProbeCard 和 DUT 作为一级分析维度。

ProbeCard 触发排名
DUT# 触发热度

优点

  • 体验最连贯，一键出全部分析
  • 指标卡第一眼就看到关键数字
  • ProbeCard/DUT 成为核心维度

缺点

  • 要重写两个组件，改动较大
C

保留旧 tab + 新增专项 tab

保留现有 yield monitor 和 JB START 不动，额外新增两个专项 tab：根因排查 和 批次汇总，分别承接 A 和 C 两类使用场景。

API目录 | yield monitor | JB START | 根因排查 ✚ | 批次汇总 ✚ | 表浏览
根因排查
Lot+Device → Slot 不良 → Bin 分布 → ProbeCard → DUT
批次汇总
Device + 时间 → 各 Lot yield% → 趋势 → 探针卡排名

优点

  • 不破坏现有功能
  • 新旧并存

缺点

  • tab 增加到6个，导航拥挤
  • 新旧 tab 功能有重叠
