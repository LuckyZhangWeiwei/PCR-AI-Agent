# AI Agent (Function Calling) Design — PCR-AI-Agent

**Date:** 2026-05-16  
**Status:** Approved  
**Scope:** Add a ReAct-loop AI agent with function calling to the existing monorepo, accessible via web interface.

---

## 1. Goals

- Natural-language queries over Yield Monitor and JB STAR data via existing `/api/v4` endpoints
- LLM automatically calls tools (list queries, aggregate queries, chart generation) using function calling
- Streaming SSE response so user sees output in real time
- ECharts charts rendered inline in the chat conversation flow
- Model / API key / API base URL configurable from the frontend Settings tab (no redeploy needed)
- Multi-turn conversation history within a session

---

## 2. Non-Goals (first version)

- No history compression (sessions are short-lived, in-memory only)
- No dispatch_workers / sub-agent parallelism
- No reflection loops / plan confirmation mode
- No persistent memory across sessions (no memory.md)
- No new npm package (integrate into existing `pcr-ai-api`)

---

## 3. Architecture Overview

```
Browser (pcr-ai-report)
  AiAgentReport.tsx
    └─ POST /api/v4/agent/chat  (SSE)
         └─ pcr-ai-api Express
              src/routes/agent.ts
                └─ src/lib/agent/
                     agentConfig.ts    # config from request body → env fallback
                     agentStream.ts    # SiliconFlow SSE streaming (node:https, no undici)
                     agentTools.ts     # 5 tool schemas + dispatch (calls internal lib fns)
                     agentLoop.ts      # ReAct loop (max 5 rounds)
                     agentHistory.ts   # Map<sessionId, Message[]>, 30-min TTL cleanup
```

All tool calls go directly to internal lib functions — no HTTP self-calls.

## 4. Backend

### 4.1 New Files

| File | Responsibility |
|------|---------------|
| `src/lib/agent/agentConfig.ts` | Merge request-body agentConfig with env vars; export `AgentConfig` type |
| `src/lib/agent/agentStream.ts` | POST to SiliconFlow `/chat/completions` with `stream:true`; yield SSE chunks via AsyncGenerator |
| `src/lib/agent/agentTools.ts` | `TOOL_SCHEMAS[]` (5 JSON schemas) + `runTool(name, args, config)` dispatcher |
| `src/lib/agent/agentLoop.ts` | `runAgentLoop(message, history, agentConfig, sseEmit)` — ReAct loop, max 5 rounds |
| `src/lib/agent/agentHistory.ts` | `getHistory(sessionId)`, `appendHistory(sessionId, msg)`, `clearHistory(sessionId)`; 30-min TTL |
| `src/routes/agent.ts` | `POST /api/v4/agent/chat` — sets SSE headers, calls agentLoop, emits events |

### 4.2 Endpoint

```
POST /api/v4/agent/chat
Content-Type: application/json

Body:
{
  "message":   string,          // user message
  "sessionId": string,          // UUID generated by frontend; creates new session if unknown
  "agentConfig": {              // all optional; falls back to env vars
    "apiKey":  string,
    "apiBase": string,
    "model":   string
  }
}

Response: text/event-stream (SSE)
```

### 4.3 SSE Event Types

```
data: {"type":"text",        "delta":"..."}
data: {"type":"tool_start",  "name":"...", "args":{...}}
data: {"type":"tool_result", "name":"...", "summary":"..."}
data: {"type":"chart",       "option":{...}}
data: {"type":"done"}
data: {"type":"error",       "message":"..."}
```

`tool_result.summary` is truncated to 200 chars for display; full result goes into LLM history.  
`chart.option` is a complete ECharts option object, ready for `echartsInstance.setOption(option)`.

### 4.4 AgentConfig Resolution

Priority: request body `agentConfig` field → env var → hardcoded default.

| Setting | Env var | Default |
|---------|---------|---------|
| apiKey | `AGENT_API_KEY` → `SILICONFLOW_API_KEY` | (required) |
| apiBase | `AGENT_API_BASE` → `SILICONFLOW_API_BASE` | `https://api.siliconflow.cn/v1` |
| model | `AGENT_MODEL` | `deepseek-ai/DeepSeek-V3` |

### 4.5 agentStream.ts

- Uses `node:https` (`rejectUnauthorized: false` matching `siliconflowChat.ts`) for SiliconFlow
- Sends `stream: true`, `stream_options: { include_usage: true }`
- Parses `data:` SSE lines, accumulates text deltas and tool_calls fragments across chunks
- Returns `AsyncGenerator<StreamChunk>` where `StreamChunk` is `{ type: "delta"|"tool_calls"|"done", ... }`
- No undici, no external HTTP library
- Node 18 compatible: uses `node:https` IncomingMessage as async iterator via `async for...of`

### 4.6 Tools

#### Tool 1: `query_yield_triggers`
Calls `parseYieldMonitorTriggerV3Query()` + Oracle/Dummy list path internally.

Parameters: `device?`, `lotId?`, `wafer?`, `hostname?`, `probeCard?`, `probeCardType?`, `pass?`, `timeFrom?`, `timeTo?`, `limit?` (default 50, max 200)

Returns: `{ count, rows: [...] }` — stored in history truncated to 3000 chars.

#### Tool 2: `aggregate_yield_triggers`
Calls v3 aggregate path internally.

Parameters: `dimensions` (comma-separated: `device`, `hostname`, `lotId`, `wafer`, `probeCard`, `pass`, `timeDay`), `groupTop?` (default 10), + all filter params from Tool 1.

Returns: `{ totalRowsMatching, groups: [...] }`

#### Tool 3: `query_jb_bins`
Calls `parseInfcontrolLayerBinsV3Query()` + Oracle/Dummy list path internally.

Parameters: `device?`, `lot?`, `slot?`, `cardId?`, `probeCardType?`, `testerId?`, `passId?`, `meslot?`, `testEndFrom?`, `testEndTo?`, `limit?` (default 50, max 200)

Returns: `{ count, rows: [...] }`

#### Tool 4: `aggregate_jb_bins`
Calls v3 aggregate path internally. Automatically ensures `bin` is present in groupBy.

Parameters: `groupBy` (comma-separated; `bin` added if missing), `groupTop?` (default 10), + all filter params from Tool 3.

Returns: `{ totalRowsMatching, groups: [...] }`

#### Tool 5: `generate_chart`
Pure computation, no DB call. Builds ECharts option JSON from provided data.

Parameters:
- `chartType`: `"bar" | "line" | "pie" | "scatter"`
- `title`: string
- `data`: `{ labels: string[], series: { name: string, values: number[] }[] }`

Returns: `{ __chartOption: EChartsOption }` — a sentinel object. `agentLoop` detects this, emits `{"type":"chart","option":{...}}` SSE event, and stores only `"[图表已生成]"` in LLM history (avoids wasting tokens on the full option JSON).

### 4.7 ReAct Loop (agentLoop.ts)

```
for round in 0..4:
  call LLM with history + TOOL_SCHEMAS
  if finish_reason == "tool_calls":
    emit tool_start events
    execute all tools (sequentially, to avoid pool exhaustion — matches REPORT_ORACLE_FANOUT_CONCURRENCY=1)
    emit tool_result events (and chart event if tool is generate_chart)
    append tool results to history
    continue
  else (finish_reason == "stop"):
    emit text events (streamed)
    emit done
    return
emit error("max rounds reached")
```

Tools executed **sequentially** (not parallel) — consistent with the existing `REPORT_ORACLE_FANOUT_CONCURRENCY = 1` constraint to avoid Oracle pool exhaustion.

### 4.8 Session History (agentHistory.ts)

```typescript
type Message = { role: "user"|"assistant"|"tool"; content?: string; tool_calls?: ...; tool_call_id?: string }

Map<sessionId, { messages: Message[], lastActive: number }>
```

- Max 40 messages per session (drop oldest user+assistant pair when exceeded)
- Cleanup timer runs every 5 minutes, removes sessions idle > 30 minutes
- `clearHistory(sessionId)` called when frontend sends new-session signal

### 4.9 System Prompt

```
你是 NXP ATTJ WaferTest 数据分析助手。
可用工具：query_yield_triggers, aggregate_yield_triggers, query_jb_bins, aggregate_jb_bins, generate_chart。

规则：
- 查到数据后，主动调用 generate_chart 生成合适的图表
- 用中文回答，数字结论要具体
- 时间范围未指定时，API 默认查最近 1 年数据
- Yield Monitor 数据来自 YMWEB_YIELDMONITORTRIGGER 表（delta_diff 类型）
- JB STAR 数据来自 INFCONTROL ⋈ INFLAYERBINLIST（PASSTYPE=TEST）
```

### 4.10 Node v18 Compatibility

- No `Array.at()` — use `arr[arr.length - 1]`
- No top-level `await` in library files — wrap in async functions
- `node:https` for HTTP — no undici
- `crypto.randomUUID` not needed server-side (sessionId comes from frontend)
- `IncomingMessage` from `node:https` implements `AsyncIterable<Buffer>` in Node 18 — `for await...of` works directly, no polyfill needed

### 5.3 AiAgentReport Redesign

**Message types in state:**
```typescript
type ChatMessage =
  | { kind: "user";   text: string }
  | { kind: "ai";     text: string; streaming: boolean }
  | { kind: "tool";   name: string; summary: string; open: boolean }
  | { kind: "chart";  option: object }
  | { kind: "error";  message: string }
```

**Sending a message:**
1. Append user message to state
2. `fetch(apiBase + "/api/v4/agent/chat", { method: "POST", body: JSON.stringify({message, sessionId, agentConfig}) })`
3. Read `response.body` as `ReadableStream`, decode with `TextDecoder`
4. Parse SSE lines (`data: {...}`) and dispatch to state:
   - `text` → append delta to current AI message
   - `tool_start` → push tool message
   - `tool_result` → update tool message summary
   - `chart` → push chart message
   - `done` → mark AI message streaming=false
   - `error` → push error message

**sessionId:** `crypto.randomUUID()` on component mount; reset on「新对话」button click.

**「新对话」button:** clears local message state + generates a new `sessionId` (`crypto.randomUUID()`). The old sessionId's server-side history naturally expires after 30 minutes. No special clear endpoint needed.

**ECharts rendering:** reuse existing `DarkChart` component (`src/components/DarkChart.tsx`) which already wraps ECharts with the dark palette theme.

## 6. Error Handling

| Scenario | Behavior |
|----------|----------|
| LLM API key missing/invalid | Emit `{"type":"error","message":"API Key 未配置或无效"}` |
| Tool call fails (Oracle error) | Tool returns error string; LLM sees it and may retry with different params or explain |
| Max rounds (5) reached | Emit `{"type":"error","message":"已达到最大推理轮数"}` |
| SSE connection dropped by client | `req.on("close")` aborts in-flight LLM request |
| Oracle pool exhaustion | Tools execute sequentially; same protection as existing report |

## 8. Out of Scope / Future

- History compression (LLM summarization when token count grows large)
- `dispatch_workers` sub-agent pattern for parallel multi-dimensional analysis
- Reflection loops (`[REFLECT]` blocks)
- Cross-session memory persistence
- Streaming tool results (current design: tool executes fully, then result emitted)
